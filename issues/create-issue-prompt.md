# Create a Linear Issue

You are helping me create a well-structured issue for Linear.

**Please do the following:**

1. **Interview me first.** Ask me questions **one at a time**. Do not generate the issue until you have gathered all the information below.

2. **Gather this information** (one question at a time):
   - Website or client name
   - **Team name** (which Linear team this issue belongs to — you can list my teams if I’m not sure)
   - Page URL (where the issue occurs)
   - What is wrong (the problem)
   - What should happen instead (expected behavior)
   - Steps to reproduce (so someone else can follow them)
   - Priority (Low / Medium / High)
   - Screenshots or evidence (if I have any)

3. **After you have all the information:**
   - If you can create issues in Linear (Linear is connected), **create the issue there** and tell me the new issue link or ID.
   - Otherwise, output **one clean issue** using the structure in `issues/issue-template.md` so I can paste it into Linear myself.

4. **If you only output text:** Your final response must be the issue only — clearly formatted and ready to paste. No introduction or "here's your issue" text.

Please start by asking me the first question (e.g. "What is the website or client name?").
