# Ecommerce & Local Business UX Audit

Audit sites and designs against research-grounded rules. Tie every finding to a **specific guideline**; avoid generic or unsupported suggestions.

## When to Use

- User asks for a UX audit, UI review, or conversion improvements
- Auditing an ecommerce or local business website (or Figma/mockups)
- Reviewing checkout flows, product pages, navigation, or account/self-service
- Preparing design feedback or CRO recommendations

## Audit Workflow

1. **Scope** — Determine project type: local business, ecommerce, or both. Identify pages/flows in scope (homepage, nav, PLP, PDP, checkout, account, mobile).
2. **Reference rules** — Use [reference-rules.md](reference-rules.md) for the full checklist and rules. Only cite rules that apply to the project type and pages in scope.
3. **Compare** — For each relevant section, compare the current UI/copy/flow to the rules. Note **gaps** (missing or wrong) and **strengths** (already aligned).
4. **Prioritize** — Call out high-impact, small changes (e.g. delivery date instead of "2–3 days", guest checkout prominence, required vs optional field labeling).
5. **Output** — Use the report structure below. Every recommendation must reference a rule; do not suggest changes that contradict the reference rules.

## Report Structure

```markdown
# UX Audit: [Site/Project Name]

**Scope:** [Local business / Ecommerce / Both] — [Pages or flows covered]

## Summary
[2–4 sentences: overall alignment, top risks, top opportunities]

## Findings

### [Section name, e.g. Checkout]
- **Finding:** [What’s wrong or missing]
  **Rule:** [Short reference to rule, e.g. "Guest checkout most prominent"]
  **Recommendation:** [Concrete, actionable fix]

### [Next section]
...

## Strengths
[What already follows the guidelines]

## Quick Wins
[Small, high-impact changes to do first]
```

## Core Principles

- **Rule-based only** — Every suggestion must map to a rule in reference-rules.md. Do not add generic "best practice" or unsupported ideas.
- **Small details matter** — Delivery date vs "2–3 days", field labels, button placement, and trust cues can be conversion-critical.
- **No harmful advice** — Do not recommend changes that conflict with the reference (e.g. forcing account creation, hiding guest checkout, omitting shipping cost near Buy).
- **Project-type fit** — Local business: emphasize footer links, in-store return/pickup, phone explanation, homepage breadth. Ecommerce: apply full checklist for nav, PLP, PDP, checkout, accounts, mobile, accessibility.
